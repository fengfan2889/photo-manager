# PhotoManager - 照片管理软件需求文档

## 1. 项目概述

### 1.1 项目背景

随着数码照片数量激增，照片管理成为刚性需求。现有解决方案（如 Google Photos、Apple Photos）依赖云端，而自托管照片管理软件（如 PhotoPrism、LibrePhotos）虽然开源，但功能复杂度过高。本项目定位为**轻量级本地照片管理器**，专注于自动化整理与灵活标注。

### 1.2 核心定位

| 维度 | 描述 |
|------|------|
| **目标用户** | 摄影师、设计师、有大量本地照片的个人用户 |
| **核心价值** | 自动化整理 + 灵活标注 + 数据可迁移 |
| **差异化** | 本地优先、JSON导出保证数据可迁移 |

## 2. 功能需求

### 2.1 图片目录自动整理

#### 2.1.1 时间优先级规则

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| P0 | EXIF 拍摄时间 | `DateTimeOriginal` 字段 |
| P1 | EXIF 修改时间 | `DateTime` 字段 |
| P2 | 文件创建时间 | 系统 `ctime` |
| P3 | 文件修改时间 | 系统 `mtime` |

> **PhotoPrism 实践**：当缺少 EXIF 时，采用文件创建/修改时间中较早者作为 fallback

#### 2.1.2 目录结构

```
输出目录/
├── 2024/
│   ├── 2024-01-01_元旦/
│   ├── 2024-01-15_日常/
│   └── 2024-12-25_圣诞节/
├── 2025/
└── unknown_date/        # 无法识别时间的图片
```

**命名规则**：`{YYYY-MM-DD}[_{事件名}]`

#### 2.1.3 整理策略

| 策略 | 说明 |
|------|------|
| `copy` | 复制到目标目录（保留原文件） |
| `move` | 移动到目标目录（不保留原文件） |
| `link` | 创建符号链接（节省空间） |

### 2.2 SQLite 数据库设计

#### 2.2.1 核心表结构

```sql
-- 照片记录表
CREATE TABLE photos (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path       TEXT NOT NULL UNIQUE,          -- 原始路径
    file_hash       TEXT NOT NULL,                  -- SHA256 哈希
    file_size       INTEGER,                        -- 字节
    file_name       TEXT NOT NULL,
    
    -- 时间信息
    taken_at        DATETIME,                       -- EXIF 拍摄时间
    taken_at_local  DATETIME,                       -- 本地化时间
    created_at      DATETIME,                       -- 文件创建时间
    modified_at     DATETIME,                       -- 文件修改时间
    
    -- EXIF 元数据
    camera_make     TEXT,
    camera_model    TEXT,
    lens_model      TEXT,
    iso             INTEGER,
    aperture        REAL,
    shutter_speed   TEXT,
    focal_length    REAL,
    photo_lat       REAL,                          -- GPS 纬度
    photo_lon       REAL,                          -- GPS 经度
    
    -- 整理信息
    organized_path  TEXT,                           -- 整理后路径
    organized_date  TEXT,                           -- 整理到的日期目录
    
    -- 标注信息
    is_portrait     INTEGER DEFAULT 0,              -- 是否人像 (0/1)
    rating          INTEGER DEFAULT 0,              -- 评级 0-5
    
    -- 缩略图
    thumb_path      TEXT,
    
    -- 元数据
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 标签表
CREATE TABLE tags (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    color       TEXT DEFAULT '#808080'
);

-- 照片-标签关联表
CREATE TABLE photo_tags (
    photo_id    INTEGER REFERENCES photos(id) ON DELETE CASCADE,
    tag_id      INTEGER REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (photo_id, tag_id)
);

-- 人像检测记录表
CREATE TABLE faces (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    photo_id        INTEGER REFERENCES photos(id) ON DELETE CASCADE,
    face_x          REAL,                          -- 人脸区域坐标
    face_y          REAL,
    face_w          REAL,
    face_h          REAL,
    embedding       BLOB,                          -- 人脸特征向量
    subject_name    TEXT,                          -- 人物名称（如已知）
    confidence      REAL                           -- 检测置信度
);

-- 整理历史表
CREATE TABLE organize_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    photo_id    INTEGER REFERENCES photos(id) ON DELETE CASCADE,
    action      TEXT NOT NULL,                     -- 'add', 'update', 'delete'
    old_path    TEXT,
    new_path    TEXT,
    timestamp   DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.2.2 索引优化

```sql
-- 时间查询优化
CREATE INDEX idx_photos_taken_at ON photos(taken_at);
CREATE INDEX idx_photos_organized_date ON photos(organized_date);

-- 人像查询优化
CREATE INDEX idx_photos_is_portrait ON photos(is_portrait);
CREATE INDEX idx_photos_rating ON photos(rating);

-- 文件哈希去重
CREATE UNIQUE INDEX idx_photos_file_hash ON photos(file_hash);
```

### 2.4 图片标注功能

#### 2.4.1 人像检测

| 功能 | 说明 |
|------|------|
| 自动检测 | 使用 ML 模型自动检测人脸 |
| 区域标记 | 记录人脸在图片中的位置 |
| 人物命名 | 支持为检测到的人脸命名 |
| 人脸聚类 | 自动将相似人脸归类 |

**技术方案**：
- 轻量级：`face_recognition` + `dlib`（Python）
- 高精度：`insightface` + ONNX

#### 2.4.2 级别标注

| 评级 | 含义 |
|------|------|
| 0 | 未评级 |
| 1 | 废片 |
| 2 | 一般 |
| 3 | 良好 |
| 4 | 优秀 |
| 5 | 杰作 |

#### 2.4.3 标签系统

- 自定义标签（支持颜色）
- 预设标签（人像、风景、建筑、宠物等）
- 批量添加/移除标签

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标 |
|------|------|
| 万张照片索引 | < 30 分钟 |
| 单张照片查询 | < 50ms |
| 缩略图生成 | < 500ms/张 |
| 启动时间 | < 3 秒 |

### 3.2 技术栈建议

```
核心语言: Python 3.10+
EXIF 处理: exifread / piexif
图片处理: Pillow
数据库: SQLite3
人脸检测: face_recognition / insightface
缩略图: Pillow (thumbnail)
JSON: json / orjson
CLI框架: Click / Typer
```

### 3.3 数据安全

- 原始文件操作默认使用复制模式（不破坏原文件）
- 数据库每日自动备份
- JSON 导出作为灾备
