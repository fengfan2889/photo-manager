## 项目概述

- **项目名称**: PhotoManager

## IPC 命令清单

### 通信协议

请求格式：
```json
{"command": "命令名", "args": {"参数": "值"}}
```

响应格式：
```json
{"success": true, "data": {...}, "id": "请求ID"}
{"success": false, "error": "错误信息", "id": "请求ID"}
```

|------|------|------|------|

| `delete-tag` | `id` | `boolean` | 删除标签 |

| `remove-tag` | `photo_id`, `tag_id` | `boolean` | 移除标签 |

### 日志模块 (Logger)

#### 日志级别定义

| 级别 | 数值 | 用途 | 输出示例 |
|------|------|------|----------|
| TRACE | 5 | 最详细（函数入口、参数） | `>>> enter organize()` |
| DEBUG | 10 | 开发调试（变量值、流程） | `file_hash = abc123` |
| INFO | 20 | 正常流程 | `Organized 100 photos` |
| SUCCESS | 25 | 成功操作 | `Photo saved: IMG_001.jpg` |
| WARNING | 30 | 警告（可继续） | `Missing EXIF, using mtime` |
| ERROR | 40 | 错误（操作失败） | `Failed to read file` |
| FATAL | 50 | 致命错误（程序终止） | `Database connection failed` |

#### 日志格式

```
{TIMESTAMP} | {LEVEL:^8} | {SOURCE:^15} | {MESSAGE} | {EXTRA}

示例:

2026-04-04 20:30:15 | DEBUG    | exif_parser   | EXIF DateTimeOriginal: 2024:06:15 14:30:00
2026-04-04 20:30:15 | INFO     | organizer     | Organized 1/100 photos
2026-04-04 20:30:15 | WARNING  | exif_parser   | Missing EXIF, fallback to file mtime
```

#### Python 后端日志 (loguru)

```python
# python/src/logger.py
from loguru import logger
import sys
from pathlib import Path

class Logger:
    _instance = None
    _log_level = "INFO"  # 默认级别
    
    @classmethod
    def init(cls, log_dir: str, level: str = "INFO"):
        """初始化日志器
        
        Args:
            log_dir: 日志文件目录
            level: 日志级别 (TRACE/DEBUG/INFO/SUCCESS/WARNING/ERROR)
        """
        cls._log_level = level.upper()
        log_file = Path(log_dir) / "photo-manager.log"
        
        # 移除默认 handler
        logger.remove()
        
        # 控制台输出（带颜色）
        logger.add(
            sys.stderr,
            level=cls._log_level,
            format="<green>{time:YYYY-MM-DD HH:mm:ss}</green> | <level>{level:^8}</level> | <cyan>{name}</cyan>.<cyan>{function}</cyan>:<cyan>{line}</cyan> | <level>{message}</level>",
            colorize=True
        )
        
        # 文件输出（完整日志）
        logger.add(
            log_file,
            level="TRACE",  # 文件记录所有级别
            rotation="10 MB",
            retention="7 days",
            format="{time:YYYY-MM-DD HH:mm:ss.SSS} | {level:^8} | {name}:{function}:{line} | {message}",
            encoding="utf-8"
        )
        
        # 错误日志单独文件
        error_file = Path(log_dir) / "error.log"
        logger.add(
            error_file,
            level="ERROR",
            format="{time:YYYY-MM-DD HH:mm:ss.SSS} | {level:^8} | {name}:{function}:{line} | {message}\n{exception}",
            encoding="utf-8"
        )
    
    @classmethod
    def set_level(cls, level: str):
        """动态设置日志级别"""
        cls._log_level = level.upper()
        # 通过环境变量传递给子进程
        import os
        os.environ["LOG_LEVEL"] = cls._log_level
    
    @classmethod
    def get_logger(cls, module: str):
        """获取模块日志器"""
        return logger.bind(name=module)

# 使用示例
log = Logger.get_logger(__name__)

log.trace("Entering function with args:", {"arg1": value1})
log.debug("Variable state:", {"hash": file_hash})
log.info("Processing photo:", {"path": file_path})
log.success("Operation completed:", {"count": count})
log.warning("Fallback triggered:", {"reason": "missing exif"})
log.error("Operation failed:", {"error": str(e)})
log.exception("Fatal error:", e)  # 自动包含 traceback
```

#### Electron 主进程日志 (electron-log)

```typescript
// electron/src/main/logger.ts
import log from 'electron-log';
import { app } from 'electron';
import path from 'path';

export class Logger {
  private static logLevel: string = 'info';
  
  static init(logDir?: string): void {
    const userDataPath = app?.getPath('userData') || process.cwd();
    const logPath = logDir || path.join(userDataPath, 'logs');
    
    // 配置文件日志
    log.transports.file.resolvePathFn = () => path.join(logPath, 'main.log');
    log.transports.file.level = 'trace';  // 文件记录所有
    log.transports.file.format = '[{y}-{m}-{d} {h}:{i}:{s}.{ms}] [{level}] {text}';
    log.transports.file.maxSize = 10 * 1024 * 1024; // 10MB
    
    // 错误日志单独文件
    log.transports.file.getFile().resolvePathFn = () => path.join(logPath, 'error.log');
    log.transports.error.level = 'error';
    
    // 控制台日志
    log.transports.console.level = this.logLevel;
    log.transports.console.format = '[{h}:{i}:{s}] [{level}] {text}';
    
    // 设置级别
    this.setLevel(this.logLevel);
    
    log.info('Logger initialized', { logPath });
  }
  
  static setLevel(level: string): void {
    this.logLevel = level.toLowerCase();
    log.transports.console.level = this.logLevel;
  }
  
  static trace(message: string, data?: object): void {
    log.transports.console.level = 'trace';
    log.info(`[TRACE] ${message}`, data || '');
  }
  
  static debug(message: string, data?: object): void {
    log.debug(message, data || '');
  }
  
  static info(message: string, data?: object): void {
    log.info(message, data || '');
  }
  
  static warn(message: string, data?: object): void {
    log.warn(message, data || '');
  }
  
  static error(message: string, error?: Error): void {
    log.error(message, error?.stack || error?.message || '');
  }
  
  static verbose(): string {
    return 'electron-log';
  }
}
```

#### React 前端日志

```typescript
// electron/src/renderer/src/utils/logger.ts
type LogLevel = 'trace' | 'debug' | 'info' | 'warn' | 'error';

interface LogEntry {
  timestamp: string;
  level: LogLevel;
  source: string;
  message: string;
  data?: object;
}

class FrontendLogger {
  private static logs: LogEntry[] = [];
  private static maxLogs = 1000;
  private static level: LogLevel = 'info';
  
  static init(level: LogLevel = 'info'): void {
    this.level = level;
    this.info('[Logger] Frontend logger initialized', { level });
  }
  
  static setLevel(level: LogLevel): void {
    this.level = level;
  }
  
  private static shouldLog(level: LogLevel): boolean {
    const levels: Record<LogLevel, number> = {
      trace: 0, debug: 1, info: 2, warn: 3, error: 4
    };
    return levels[level] >= levels[this.level];
  }
  
  private static log(level: LogLevel, source: string, message: string, data?: object): void {
    if (!this.shouldLog(level)) return;
    
    const entry: LogEntry = {
      timestamp: new Date().toISOString(),
      level,
      source,
      message,
      data
    };
    
    this.logs.push(entry);
    if (this.logs.length > this.maxLogs) {
      this.logs.shift();
    }
    
    const prefix = `[${level.toUpperCase()}] ${source}`;
    const args = data ? [prefix, message, data] : [prefix, message];
    
    switch (level) {
      case 'trace':
      case 'debug':
        console.debug(...args);
        break;
      case 'info':
        console.info(...args);
        break;
      case 'warn':
        console.warn(...args);
        break;
      case 'error':
        console.error(...args);
        break;
    }
    
    // 可选：发送到主进程（记录到文件）
    if (window.electronAPI?.logToFile) {
      window.electronAPI.logToFile(entry);
    }
  }
  
  static trace(source: string, message: string, data?: object): void {
    this.log('trace', source, message, data);
  }
  
  static debug(source: string, message: string, data?: object): void {
    this.log('debug', source, message, data);
  }
  
  static info(source: string, message: string, data?: object): void {
    this.log('info', source, message, data);
  }
  
  static warn(source: string, message: string, data?: object): void {
    this.log('warn', source, message, data);
  }
  
  static error(source: string, message: string, error?: Error): void {
    this.log('error', source, message, { error: error?.message, stack: error?.stack });
  }
  
  static getLogs(): LogEntry[] {
    return [...this.logs];
  }
  
  static clearLogs(): void {
    this.logs = [];
  }
}

export const logger = FrontendLogger;
```

## 设计原则

### 1. 表命名

```
{模块}_{内容描述}

规则：
- 模块前缀：photo / sys
- 内容描述：用下划线分隔，体现业务含义
- 允许多段式命名

示例：
photo_info              → 照片信息
photo_face_detect       → 人脸检测
photo_face_subject      → 人物主题
photo_photo_tag         → 照片标签关联
sys_setting             → 系统设置
sys_log_operation       → 操作日志
```

### 2. 字段命名

```
snake_case（小写下划线）

规则：
- 全部小写
- 单词间用下划线
- 时间字段用 _at 后缀
- 布尔字段用 is_ / has_ 前缀

示例：
file_path, taken_at, is_portrait, created_at
```

### 3. 公共字段

所有表必须包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键，自增 |
| `created_at` | DATETIME | 创建时间 |

---

| 决策 | 原因 | 备选方案 |
|------|------|----------|
| Electron + Python | 参考 fin-agent-desktop，兼顾桌面体验和 Python 生态 | 纯 PyQt / 纯 Web |
| SQLite 存储 | 轻量、无依赖、支持 SQL 查询 | PostgreSQL, MySQL |
| JSON 导出 | 保证数据可迁移性 | 仅数据库 |
| electron-vite | 参考 fin-agent-desktop，开发体验好 | electron-forge |
| Tailwind CSS | 快速开发、参考项目已用 | styled-components |
| 时间 fallback | 兼容无 EXIF 的截图/网图 | 强制要求 EXIF |
